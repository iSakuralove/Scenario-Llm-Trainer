# 102. 排查工坊 Agent 默认不释线索决策升级

## 任务目标

- 将当前工作区里的旧版排查 Agent 决策链，从 `off_track -> guided_redirect -> release clue` 改为“默认不释线索”。
- 保持后端内部改造，不新增前端展示，不修改 HTTP 输出 schema，不新增数据库表。
- 让无效、偏题、聊天式输入只做纠偏，不再机械发放 surface clue。

## 实现内容

- 重写 `SemanticGate` 的意图分类，新增并启用：
  - `broad_probe`
  - `evidence_probe`
  - `hypothesis`
  - `answer_guess`
  - `repeat_probe`
  - `chatty_off_topic`
- 收紧语义决策边界：
  - 只有 `relevant / evidence_probe` 命中 clue 才能 `release_clue`
  - 只有 `relevant / broad_probe` 才能进入 `guided_redirect`
  - `off_track / chatty_off_topic / repeat_probe / answer_guess` 不再通过语义层直接释放线索
- 重写 `diagnostic.go` 中间策略路由：
  - `off_track -> request_rephrase`
  - `chatty_off_topic -> humorous_redirect`
  - `answer_guess/hypothesis -> ask_for_evidence`
  - `broad_probe -> narrow_scope`
  - `repeat_probe -> repeat_redirect`
- 为 `find_triggered_clue` 增加意图门禁，阻止非排查型输入仅凭关键词触发 `releaseClue`
- 引入运行时 `DiagnosticContext`，支持重复追问识别和排查维度识别
