# 100.GitHub源码分支瘦身与交付包安装

## 任务目标

基于 `交付/situational-teaching-external-run-final.zip` 创建适合推送到 GitHub 的干净源码分支，避免仓库继续混入图片、压缩包、交付产物和临时生成物。

## 实现内容

- 在隔离工作树 `.worktrees/slim-github-source` 中创建 orphan 分支 `codex/slim-github-source`，避免继承旧历史中的大文件对象。
- 解压 `situational-teaching-external-run-final.zip`，安装其中的源码、配置、脚本和 README。
- 删除 zip 内自带的前端图片/SVG 静态资产，并移除 `frontend/index.html` 中的 favicon 引用。
- 明确排除以下内容：
  - 所有图片文件：`*.png`、`*.jpg`、`*.jpeg`、`*.gif`、`*.webp`、`*.svg`、`*.ico` 等。
  - 所有压缩包：`*.zip`、`*.7z`、`*.rar`、`*.tar`、`*.tar.gz`、`*.tgz`。
  - `交付/` 目录。
  - `img/` 目录。
  - `docs/report-figures/generated/` 目录。
  - 音视频二进制产物：`*.mp3`、`*.wav`、`*.mp4`、`*.mov`、`*.avi`。
- 保留源码、测试、配置、脚本和必要文档，作为 GitHub 仓库的源码主体。

## 风险与边界

- 该分支是源码分支，不包含图片素材和交付产物；如果报告或演示需要图片，应通过 Release 附件、网盘或单独资产仓库管理。
- 因为使用 orphan 分支，推送到远程新分支不会携带旧历史中的大文件对象；但如果强制覆盖远程默认分支，需要确认团队协作风险。
- 当前主工作区仍保留原有未提交开发改动，本任务未在主工作区覆盖这些文件。

## 验证步骤

本地验证记录：

- `go test ./...`：通过。
- `npm --prefix frontend run build`：通过。隔离工作树内直接安装依赖多次超时，最终使用主工作区已有 `frontend/node_modules` 的临时目录联接完成构建验证；该目录被 `.gitignore` 忽略，不进入提交。
- 提交树检查：确认将提交的源码集合不包含图片、zip、交付目录、`img/`、`docs/report-figures/generated/`、`node_modules` 或 `dist`。

1. 验证分支不含图片、压缩包和音视频文件：

   ```powershell
   Get-ChildItem -Recurse -File -Include *.png,*.jpg,*.jpeg,*.gif,*.webp,*.bmp,*.tif,*.tiff,*.svg,*.ico,*.zip,*.7z,*.rar,*.tar,*.tgz,*.mp3,*.wav,*.mp4,*.mov,*.avi
   ```

   预期：无输出。

2. 验证明确排除目录不存在：

   ```powershell
   Test-Path -LiteralPath '交付'
   Test-Path -LiteralPath 'img'
   Test-Path -LiteralPath 'docs/report-figures/generated'
   ```

   预期：均为 `False`。

3. 验证 `.gitignore` 命中策略：

   ```powershell
   git check-ignore -v '交付/example.zip' 'img/example.png' 'docs/report-figures/generated/example.png' 'frontend/src/assets/example.svg'
   ```

   预期：每一项都由 `.gitignore` 命中。

4. 验证源码可构建：

   ```powershell
   npm --prefix frontend run build
   cd backend
   go test ./...
   ```

   预期：前端构建和后端测试均通过。

5. 推送源码分支：

   ```powershell
   git push -u origin codex/slim-github-source
   ```

   预期：远程 GitHub 出现不含大文件历史的新源码分支。
